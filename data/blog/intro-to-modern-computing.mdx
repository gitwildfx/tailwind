### ü§ñ Module 6: APIs & AI Integration ‚Äì Ask the AI

**Objective:** Learn to connect to external APIs like OpenAI‚Äôs Chat API.

**Key Concepts:**
- Fetch API  
- Async/await  
- Error handling  
- JSON structure

``` js
// api/ask.js
export default async function handler(req, res) {
  if (req.method !== 'POST') {
    return res.status(405).json({ error: 'Method not allowed' });
  }

  const { prompt } = req.body;
  if (!prompt) return res.status(400).json({ error: 'Prompt is required' });

  try {
    const response = await fetch('https://api.openai.com/v1/chat/completions', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${process.env.OPENAI_API_KEY}` // secure key from Vercel env
      },
      body: JSON.stringify({
        model: "gpt-3.5-turbo",
        messages: [{ role: "user", content: prompt }],
        max_tokens: 150,
        temperature: 0.7
      })
    });

    const data = await response.json();
    res.status(200).json(data);
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
}
```

**Frontend Changes:**

Then, in your Tower project JS, you call your server endpoint instead of OpenAI directly:

``` js
const res = await fetch("/api/ask", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ prompt: input })
});

const data = await res.json();
aiResult.textContent = data.choices[0].message.content.trim();
```

**‚úÖ Summary:**

-Frontend never sees the API key.  
-Works with Vercel environment variables (OPENAI_API_KEY).  
-Completely secure for production.  

*Warning:*

-Do not replace your frontend code with process.env.OPENAI_API_KEY directly.  
-Instead, move your OpenAI call to a serverless function and call that from the frontend.
